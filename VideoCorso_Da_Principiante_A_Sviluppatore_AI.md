## MODULO 4: COME FUNZIONA IL WEB

### Video 4.1: Client e Server
- **Client**: Il browser che richiede pagine
- **Server**: Il computer che risponde con le pagine
- **Analogia**: Ristorante (cliente ordina, chef cucina, cameriere porta)
- **Esempio pratico**: Mostrare una richiesta HTTP

### Video 4.2: URL e HTTP
- **URL**: Indirizzo di una risorsa web
- **Metodi HTTP**: GET (leggere), POST (creare), PUT (modificare), DELETE (eliminare)
- **Status code**: 200 (OK), 404 (Non trovato), 500 (Errore server)
- **Esercizio**: Analizzare le richieste di un sito web

### Video 4.3: API - Le Interfacce di Comunicazione
- **API**: Application Programming Interface
- **REST API**: Convenzioni per le richieste web
- **Esempio**: API del meteo, API di YouTube
- **Esercizio**: Chiamare un'API pubblica
